# Plano Executivo 30/60/90

## Objetivo
- estabilizar o produto
- reduzir risco operacional
- fechar base para monetização

## 0-30 dias
- concluir rotação real de segredos em produção
- fechar matriz de permissões e aplicar padrão único nos módulos críticos
- remover fluxo de token por query string
- restringir fallback de admin para bootstrap controlado ou development
- ativar branch protection com checks obrigatórios
- rodar smoke E2E agendado com credenciais reais no GitHub Actions
- ampliar testes para:
  - auth
  - financeiro
  - leads
  - customers
  - campanhas
- revisar índices e queries dos módulos:
  - financeiro
  - campanhas
  - logs/auditoria

## 31-60 dias
- quebrar páginas gigantes do frontend em módulos menores
- extrair `Program.cs` em extensões por concern
- introduzir cache Redis para permissões, catálogos e dashboards
- introduzir workers/filas para campanhas, imports e IA
- criar staging operacional confiável
- consolidar dashboards operacionais:
  - erros
  - latência
  - filas
  - integrações

## 61-90 dias
- implementar billing e limites por plano
- introduzir segregação forte por conta/empresa
- lançar portal do cliente
- empacotar ofertas comerciais por nicho
- adicionar métricas de produto e funil de conversão

## Critérios de sucesso
- nenhum segredo ativo versionado
- smoke crítico rodando automaticamente
- 100% dos módulos sensíveis com autorização padronizada
- redução de incidentes por auth/permissão
- oferta comercial com pricing e onboarding definidos
