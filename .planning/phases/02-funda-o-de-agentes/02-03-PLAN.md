    Create `AgentOrchestratorService.cs`. Inject: `IEnumerable<IAgentHandler> handlers`, `IAnthropicChatClient anthropic`, `IAiChatRepository conversationRepo`, `IAgentPendingActionRepository pendingRepo`, `IUnitOfWork uow`, `ILogger<AgentOrchestratorService>`.     Constants: `private const string DefaultModel = "claude-sonnet-4-5"; private const int DefaultMaxTokens = 2048;`.

    ChatAsync:
    1. If `!_handlers.TryGetValue(agentType, out var handler)` → `Result.Failure<AgentChatResponse>(new Error("NotFound", "Tipo de agente não suportado."))`.
    2. Validate request.Message not empty → Error "Validation".
    3. Load or create conversation: if request.ConversationId.HasValue → `conversationRepo.GetWithMessagesAsync(id, userId, ct)`; if null return Error "NotFound". Else create `new AiConversation(userId, "Nova conversa", model, system)`, call `conv.SetAgentType(agentType)`, `conversationRepo.AddAsync(conv, ct)`.
    4. Build AnthropicMessage list from conv.Messages ordered by CreatedAt (role/content) + the new user message. system = handler.BuildSystemPrompt(userId, request.Context).
    5. Persist the user message (`new AiChatMessage(conv.Id, "user", request.Message)` via AddMessageAsync) then SaveChangesAsync.
    6. tools = handler.GetTools(). If tools.Count == 0 → call `anthropic.CompleteAsync(req, ct)`, persist assistant message with usage + CalculateCost, SaveChanges, return AgentChatResponse(conv.Id, completion.Text, PendingAction: null, usage).
    7. Else → toolDefs = tools.Select(t => new AnthropicToolDefinition(t.Name, t.Description, t.InputSchema)); call `anthropic.CompleteWithToolsAsync(req, toolDefs, ct)`. Switch on result:
       - TextResponse → persist + return AgentChatResponse with Reply, PendingAction null.
       - ToolUseRequest tu → find `tool = tools.First(t => t.Name == tu.ToolName)`. Call `var toolResult = await tool.ExecuteAsync(tu.ToolInput, userId, ct);`. If `toolResult is AgentToolResult.PendingConfirmation pc` → create `var pending = new AgentPendingAction(userId, conv.Id, agentType, tu.ToolName, tu.ToolUseId, pc.ActionLabel, pc.SerializedPayload);` → `pendingRepo.AddAsync(pending, ct)`; SaveChangesAsync; return AgentChatResponse(conv.Id, Reply: null, PendingAction: new PendingActionDto(pending.Id, tu.ToolName, pc.ActionLabel), usage). If `toolResult is AgentToolResult.Ok ok` (read tool executed inline) → continue conversation by sending tool_result as a follow-up user message via CompleteAsync (compose `<tool_result>` text) and return TextResponse-style reply — keep simple: persist + return ok.Content as Reply. If Failure → Error "Validation".

    ConfirmAsync:
    1. Route handler by agentType (Error "NotFound" if missing).
    2. `var pending = await pendingRepo.GetPendingByIdAsync(request.PendingActionId, userId, ct);` if null → Error "NotFound". If `!pending.IsPending` (executed/cancelled/expired) → Error "Validation" with message "Ação pendente expirada ou já processada." (maps to 400 — Pitfall 5).
    3. Find the tool: `var tool = handler.GetTools().FirstOrDefault(t => t.Name == pending.ToolName);` if null → Error "Validation".
    4. `var execResult = await tool.ExecuteConfirmedAsync(pending.Payload, userId, ct);` if Failure → Error "Validation". On Ok: `pending.MarkExecuted(); await pendingRepo.UpdateAsync(pending, ct);`.
    5. Load conversation, persist an assistant message confirming (use ok.Content as reply text), SaveChangesAsync. Return AgentChatResponse(pending.ConversationId, ok.Content, PendingAction: null, usage with zeros if no LLM call). (Optional: a follow-up CompleteAsync to phrase confirmation — keep minimal in Phase 2; returning ok.Content is acceptable.)

    ListConversationsAsync: `conversationRepo.GetPagedByUserAsync(userId, page, pageSize, includeArchived:false, agentType: agentType, ct)` then map to AgentConversationListItemDto (only conversations of that AgentType returned — this proves AiChat conversations with AgentType==null are excluded, ORCH-05).
    GetConversationAsync: `conversationRepo.GetWithMessagesAsync(...)`; if null or `conv.AgentType != agentType` → Error "NotFound"; map to ConversationDetailDto.

    Create test file `AgentOrchestratorServiceTests.cs` with Moq for all dependencies and a fake handler list. Cover the six behaviors above. For the "write tool, no write before confirm" test, mock `CompleteWithToolsAsync` to return a ToolUseRequest, register a fake write tool whose ExecuteConfirmedAsync is the only path that touches a `Mock<IFakeWriter>`; after ChatAsync assert `pendingRepo.Verify(r => r.AddAsync(It.IsAny<AgentPendingAction>(), ...), Times.Once)` and `writer.Verify(..., Times.Never)`. For ConfirmAsync test, set the pending repo to return a pending action and assert writer Times.Once + MarkExecuted via UpdateAsync Times.Once.
